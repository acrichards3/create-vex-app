vexkit spec dashboard

The dashboard is a local web UI for editing .vex logic trees, running a spec-first workflow, and chatting with an assistant that can change files in the project you started it from.

Starting the server

Run the dashboard from your project root (the repo you want the assistant to work in):

bun run vexkit:spec

Default URL: http://localhost:8888/ . The process uses the current working directory as the project root for file APIs and the assistant.

Six-step workflow

  1. Describe — You explain what you want. The first successful chat reply moves you toward Spec.
  2. Spec — Assistant may create or edit only .vex files (writes under .vexkit/ are blocked).
  3. Approve — You approve each function in the focused .vex file, or use Approve all.
  4. Build — Continuing from Approve runs codegen-spec for the focused .vex , then the assistant implements code and paired .spec.ts files. The assistant must not modify .vex files.
  5. Verify — Runs format check, ESLint, typecheck, and the paired spec test for the current .vex when a path is set. Chat is disabled here.
  6. Done — Mark complete; you can restart the workflow from the UI.

Workflow state is stored in .vexkit/workflow.json under the project root.

Assistant backends

You can use either the Cursor CLI agent (ACP) or an OpenAI-compatible HTTP API with repo tools—not both at once for a given request.

Cursor agent (recommended for Cursor users)

  • Install the agent CLI and sign in ( agent login ) per Cursor’s docs.
  • Set VEXKIT_USE_CURSOR_AGENT=1 and CURSOR_API_KEY in the environment (for example in a root .env file when you launch with Bun from that directory).
  • If the dashboard cannot find agent on PATH , set VEXKIT_CURSOR_AGENT_BIN to the full path (output of which agent ).
  • The dashboard resolves ~/.local/bin/agent and ~/.cursor/bin/agent when those files exist, even if PATH is minimal.

OpenAI-compatible chat

  • Set VEXKIT_CHAT_API_KEY .
  • Optional: VEXKIT_CHAT_BASE_URL (default OpenAI API), VEXKIT_CHAT_MODEL (default gpt-4o-mini ).

If Cursor mode is off and no chat key is set, chat requests return an error until you configure one of the backends.

Environment variables

Variable Purpose
VEXKIT_USE_CURSOR_AGENT Set to 1 to use the Cursor agent acp subprocess.
CURSOR_API_KEY Required for the dashboard to select Cursor mode (in addition to the flag above).
VEXKIT_CURSOR_AGENT_BIN Optional full path to the agent executable.
VEXKIT_CHAT_API_KEY OpenAI-compatible API key when not using Cursor mode.
VEXKIT_CHAT_BASE_URL Optional API base URL for OpenAI-compatible chat.
VEXKIT_CHAT_MODEL Optional model name for OpenAI-compatible chat.
VEXKIT_MCP_COMMAND_JSON Optional MCP configuration for extra tools in the OpenAI tool loop.

Streaming and chat behavior

  • Chat streams NDJSON events (delta, done, error) to the browser.
  • The workflow advances from Describe to Spec only after a successful assistant response (not after a failed HTTP request).
  • GET /api/assistant/status reports whether Cursor or OpenAI mode is active and related flags.

Verify pipeline

POST /api/workflow/verify-run runs, in order:

  1. bun run format:check
  2. bun run lint:eslint
  3. bun run typecheck
  4. If currentVexPath is set in workflow state: bun test on the paired .spec.ts (fails if the spec still contains it.todo).

Your repo’s root package.json must define those scripts for verify to succeed.

Security

The assistant runs on your machine with permission to modify the project tree (subject to phase rules and tool permissions). Do not expose the dashboard to untrusted networks or the public internet without authentication.

Troubleshooting

  • Executable not found: agent — Set VEXKIT_CURSOR_AGENT_BIN or ensure ~/.local/bin/agent exists (see above).
  • 503 on chat — Set Cursor env vars or VEXKIT_CHAT_API_KEY, restart the server, and confirm /api/assistant/status.
  • Stepper vs Describe mismatch — Clear saved UI state: remove key vexkit.dashboard.view.v1 from local storage for this origin, reload.
