Assistant
Describe

Tell the assistant what you want—one iteration can touch several .vex files. After you send your first message, the workflow moves to Spec so the model can shape the trees. Open files in the sidebar as needed; the current file is just the focused tree.

Spec

The graph is the source of truth. In spec phase, click any node in the tree to edit its label; changes are saved to the .vex file. In build phase the agent cannot edit .vex (you can still edit on disk if needed).

Approve

Approve each function tree before build. The agent cannot change .vex after you continue to build.

Function approvals
Build

Paired .spec.ts files are emitted when you continue from Approve. The assistant implements code and tests; it must not modify .vex files. Use Back to spec if requirements need to change.

Verify

Confirm the spec structure matches the .vex file, then run the paired tests.

Done

When structure checks and tests pass with no it.todo, mark the workflow done and start the next feature when you are ready.
